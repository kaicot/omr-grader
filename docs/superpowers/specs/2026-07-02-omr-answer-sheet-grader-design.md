# OMR 답안지 채점 프로그램 설계

## 배경 / 목적

동명대학교 작업치료학과에서 사용하는 5지선다 OMR 답안지(`OCR100.pdf` 양식, 최대 100문항 + 8자리 학번)를 평판 스캐너로 스캔한 이미지에서 마킹을 자동 인식하고, 정답표와 대조해 채점 결과를 엑셀로 만드는 프로그램.

## 입력

- **답안지**: 학생당 1장. 다음 두 형식이 섞여 들어올 수 있음
  - 개별 이미지 파일 (jpg/png), 폴더에 여러 장
  - 여러 장이 한 파일로 묶인 멀티페이지 PDF (페이지당 학생 1명)
- **정답표**: 별도 xlsx/csv 파일. 문항번호, 정답(1~5) 2열.
  - **이 파일의 행 수가 곧 이번 시험의 문항 수** (예: 50행이면 1~50번만 인식/채점, 51~100번 마킹칸은 무시). 별도 "문항 수" 설정 UI 불필요.

## 핵심 인식 방식

`OCR100.pdf`는 스캔 이미지가 아니라 벡터 PDF이므로, PyMuPDF로 문자 좌표를 직접 추출하면 모든 마킹칸(①~⑤, 학번 0~9)의 정확한 위치를 계산으로 얻을 수 있음 (실측 결과 문항 5칸 간격 19.44pt로 균일한 격자). 이를 근거로:

1. **템플릿**: 격자 원점 + 칸 간격 + 줄 간격 등 소수의 파라미터로 100문항×5지선다 + 학번 8자리×10행 전체 마킹칸 좌표를 계산 (좌표 500여 개를 하드코딩하지 않음).
2. **정렬**: 스캔 이미지에서 답안표 테두리(굵은 선 사각형)를 컨투어로 검출 → 투시 변환으로 기준 크기에 맞춤. 스캐너 안에서 종이가 살짝 삐뚤어지거나 밀린 경우를 보정.
3. **마킹 판정**: 정렬된 이미지에서 템플릿 좌표의 각 칸을 샘플링해 채움 정도(어두운 픽셀 비율)를 비교. 한 문항의 5칸 중 뚜렷하게 채워진 칸이 하나면 그 값을 답으로 채택.

검토 후 기각한 방식:
- 고정 픽셀좌표(정렬 없음): 스캔 시 종이 위치가 조금만 틀어져도 오인식 — 채점 도구로 부적합해 기각.
- 범용 원 검출(Hough Circle 등 정렬 불필요 방식): 이미 정확한 벡터 템플릿이 있으므로 불필요한 복잡도 — 기각.

## 애매한 마킹 처리

- **미응답**(5칸 모두 안 채워짐) / **중복 마킹**(2칸 이상 채워짐): 정답 여부와 무관하게 항상 오답(0점) 처리하되, 엑셀에 값 대신 `미응답` / `중복(n,m)` 으로 표시하고 "확인필요" 열에 해당 문항번호를 남김.
- 학생이 중간 문항까지만 답하고 이후를 비워둔 경우도 **동일한 규칙을 문항별로 그대로 적용** — "몇 번부터 미응답인지" 같은 별도 요약 로직은 만들지 않음.
- 학번 인식이 애매한 경우도 같은 방식으로 확인필요 처리.
- 정렬(스캔본에서 답안표 테두리 검출) 자체가 실패한 파일은 채점하지 않고 별도로 "정렬실패" 목록에 남기며, 나머지 파일 처리는 계속 진행.

## 출력

### 엑셀 (학생별 1행)

| 학번 | Q1 | Q2 | ... | Qn (정답표 문항 수만큼) | 점수 | 오답문항 | 확인필요 |
|---|---|---|---|---|---|---|---|
| 12345678 | 3 | 1(오답) | ... | 5 | 76 | 2,15,41 | - |
| 23456789 | 3 | 미응답 | ... | 중복(2,4) | 68 | 5,20 | Q2, Q41 |

- 오답 셀은 색으로 구분.
- "정렬실패" 파일들은 별도 시트(또는 별도 목록)에 파일명만 나열.

### 디버그 오버레이 이미지

- 채점 실행 시 `debug/` 폴더에 학생 파일명 그대로, 원본 스캔 이미지 위에 프로그램이 인식한 마킹 위치를 원으로 표시한 이미지를 함께 저장.
- 목적: "확인필요" 플래그가 붙은 학생만 골라 원본 답안지 실물 없이도 빠르게 오인식 여부를 눈으로 검증.

## 실행 방식

- **tkinter GUI** (표준 라이브러리, 추가 설치 불필요): 답안지 폴더/파일 선택, 정답표 파일 선택, "채점 시작" 버튼, 진행 로그.
- 단일 스크립트(`omr_grader.py`)로 구성 — GUI와 인식/채점 로직을 별도 모듈로 나눌 만큼 규모가 크지 않음.
- **실제 실행 환경은 이 PC가 아니라 학과사무실 PC (Python 미설치)**이므로, 개발은 이 PC(Python 3.13 + 필요 라이브러리 기설치)에서 하고, PyInstaller(`--onefile`)로 `omr_grader.exe` 하나를 빌드해 학과사무실 PC로 복사해 더블클릭 실행.
  - 정식 설치 프로그램(Inno Setup 등)은 이번 용도에 과함으로 채택하지 않음 — 단일 exe 복사/실행으로 충분.
  - onefile 특성상 실행할 때마다 압축 해제로 수 초의 시작 지연이 있을 수 있음 (opencv/numpy/pandas 포함으로 용량이 큼) — 감수 가능한 수준으로 판단.

## 에러 처리

- 답안표 테두리 검출 실패(정렬 실패) → 해당 파일 건너뛰고 엑셀에 목록으로 표시, 전체 실행은 계속.
- 정답표 파일이 비어있거나 형식이 예상과 다름 → GUI에 에러 메시지 표시 후 실행하지 않음.

## 테스트

- `OCR100.pdf`를 이미지로 렌더링한 뒤 임의의 칸 몇 개를 실제로 칠한 "가짜 마킹 스캔" 테스트 이미지를 만들어, 인식 결과가 실제로 칠한 칸과 정확히 일치하는지 확인하는 자체 검증 스크립트 하나 (assert 기반, 별도 테스트 프레임워크 없음).

## 범위 밖 (하지 않는 것)

- 정답표 자동 채점 이외의 통계/분석 기능 없음.
- 문항 수를 GUI에서 별도로 입력받는 기능 없음 (정답표 행 수로 자동 결정).
- 미응답 패턴 요약(예: "41번부터 미응답") 같은 부가 분석 없음 — 문항별 플래그로 충분.
