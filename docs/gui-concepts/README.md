# GUI 방향 시안

세 시안은 같은 핵심 화면을 서로 다른 우선순위로 표현한 비교용 이미지다.

| 방향 | 중심 가치 | 적합한 사용 장면 | 주의할 점 |
|---|---|---|---|
| [A. 차분한 검사 데스크](./방향-A-차분한-검사-데스크.png) | 기존 프로그램과의 연속성, 편안한 검토 | 현재 채점 흐름을 유지하면서 이력·재열람을 추가할 때 | 문항 통계와 변경 이력이 상대적으로 덜 강조됨 |
| [B. 증거 중심 검토 콘솔](./방향-B-증거-중심-검토-콘솔.png) | 원본·인식 결과·기준 버전·변경 전후 비교 | 재분석·재채점과 검토 큐가 핵심일 때 | 정보량이 많아 초기 사용자가 익숙해지는 시간이 필요함 |
| [C. 교수용 요약 워크스페이스](./방향-C-교수용-요약-워크스페이스.png) | 시험 요약, 학생 결과, 문항 분석 | 교수의 결과 보고와 수업 개선이 핵심일 때 | 저수준 인식 증거를 상세 화면으로 잘 연결해야 함 |

## 현재 선택안: C + B 결과 상세

전체 앱 구조는 C를 기본으로 확정한다. `결과 상세` 화면에는 B의 증거 중심 검토 흐름을 결합한다.

- C의 밝은 교수용 요약 화면, 최근 채점 목록, 문항 분석 구조
- B의 원본 이미지·인식 결과·정답키 비교 영역
- 답안 선택을 직접 수정하는 수동 채점 영역
- 수정 전후 점수와 변경 문항을 보여주는 적용 전 미리보기
- 수정 사유, 이전 값, 새 값, 검토 시각을 남기는 검토 이력
- `수정 적용`과 `취소`를 분리해 실수로 즉시 덮어쓰지 않도록 함

## 선택 방법

1. 세 이미지를 실제 모니터에서 나란히 비교한다.
2. 기본 구조는 C, 결과 상세는 B의 수동 채점 흐름으로 본다.
3. 첫 구현 기능을 하나 선택한다: `이력·재열람`, `재분석·재채점`, `검토 큐`, `문항 분석 리포트`.

첫 구현 기능 후보:

- `이력·재열람`
- `재분석·재채점`
- `검토 큐`
- `문항 분석 리포트`

시안은 구현 전 논의를 위한 구조·밀도·색상 방향이며, 실제 UI 구현 시에는 현재 `DESIGN.md`와 Windows Tkinter 제약에 맞춰 세부 조정한다.
